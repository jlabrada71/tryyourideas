# Design Brief: Try Your Ideas Platform

## Overview

Create a complete design system and home page design for **Try Your Ideas** - a collaborative platform that connects entrepreneurs with innovative ideas to skilled developers and investors. The platform democratizes access to development resources and investment capital.

**Tagline:** "Transform Your Ideas Into Reality - Connect with Developers and Investors"

---

## Brand Identity Requirements

### Core Values to Communicate Visually
1. **Transparency** - Open, honest, clear visual hierarchy
2. **Collaboration** - Connected elements, partnership imagery, unified components
3. **Innovation** - Modern, forward-thinking, bold but approachable
4. **Accessibility** - Inclusive design, clear readability, intuitive navigation
5. **Trust** - Professional, secure feeling, established credibility

### Brand Personality
- Professional yet approachable
- Innovative but not intimidating
- Trustworthy and established
- Energetic and optimistic
- Global and inclusive

### Target Audience Visual Preferences
1. **Entrepreneurs (25-55 years)**: Clean, inspiring, action-oriented
2. **Developers**: Technical credibility, modern aesthetics, efficiency-focused
3. **Investors**: Professional, data-driven, trustworthy, premium feel

---

## Design System Requirements

### 1. Color Palette

Design a comprehensive color system including:

**Primary Colors**
- Main brand color (should convey innovation, trust, and energy)
- Consider blues, purples, or teals as base - avoid overly corporate or startup-cliche colors

**Secondary Colors**
- Accent color for CTAs and highlights
- Supporting color for variety

**Semantic Colors**
- Success (for completed milestones, positive metrics)
- Warning (for deadlines, attention needed)
- Error (for validation, critical alerts)
- Info (for tips, guidance)

**Neutral Palette**
- Full grayscale spectrum (50-900)
- Background variations (light mode primary)
- Text colors (primary, secondary, muted, disabled)
- Border colors

**User Type Colors** (for visual differentiation)
- Creator/Entrepreneur badge color
- Developer badge color
- Investor badge color
- Mentor/Advisor badge color

### 2. Typography

**Font Selection**
- Primary font family (headings) - modern, distinctive, readable
- Secondary font family (body) - highly legible, works well at small sizes
- Monospace font (for code snippets, technical details)

**Type Scale**
- Display (hero headings): size, weight, line-height
- H1-H6 hierarchy
- Body large, body default, body small
- Caption, overline, label sizes
- Button text specifications

**Font Weights**
- Define usage for: Light, Regular, Medium, Semibold, Bold

### 3. Spacing System

- Base unit (recommend 4px or 8px)
- Spacing scale: xs, sm, md, lg, xl, 2xl, 3xl, 4xl
- Component internal padding standards
- Section spacing standards
- Grid gutters

### 4. Grid & Layout

- 12-column grid system
- Container max-widths (sm, md, lg, xl, 2xl)
- Breakpoints for responsive design:
  - Mobile: 320px - 639px
  - Tablet: 640px - 1023px
  - Desktop: 1024px - 1279px
  - Large Desktop: 1280px+
- Section vertical rhythm

### 5. Component Library Specifications

**Buttons**
- Primary, Secondary, Tertiary, Ghost variants
- Sizes: sm, md, lg
- States: default, hover, active, focus, disabled, loading
- Icon button variants
- Button groups

**Form Elements**
- Text inputs (with validation states)
- Textareas
- Select dropdowns
- Checkboxes and radio buttons
- Toggle switches
- Search inputs
- File upload areas
- Form labels and helper text
- Error message styling

**Cards**
- Project card (showing: title, creator, skills needed, funding goal, progress, team size)
- Developer card (showing: avatar, name, skills, rating, availability, hourly rate)
- Investor card (showing: avatar, name, investment range, portfolio focus)
- Milestone card
- Stats card

**Navigation**
- Main navigation bar (desktop)
- Mobile navigation (hamburger menu)
- User dropdown menu
- Breadcrumbs
- Tab navigation
- Sidebar navigation (for dashboards)

**Data Display**
- Tables (with sorting, pagination)
- Lists (ordered, unordered, with icons)
- Badges and tags (skill tags, status badges, user type badges)
- Progress bars and indicators
- Avatars (sizes, status indicators, group avatars)
- Rating displays (stars)
- Statistics and metrics displays

**Feedback & Overlays**
- Modals/dialogs
- Toasts/notifications
- Tooltips
- Alerts (info, success, warning, error)
- Empty states
- Loading states (skeletons, spinners)

**Rich Content**
- Blockquotes
- Code blocks
- Callout boxes
- Timeline/milestone displays
- Charts placeholder specifications

### 6. Iconography

- Icon style (outlined, filled, or duotone)
- Icon sizes (16px, 20px, 24px, 32px)
- Key icons needed:
  - Navigation (home, search, notifications, settings, menu)
  - User types (lightbulb for creators, code for developers, chart for investors)
  - Actions (add, edit, delete, share, bookmark, filter)
  - Status (check, warning, error, info, clock)
  - Social (GitHub, LinkedIn, Twitter)
  - Features (projects, team, milestone, funding, message)

### 7. Effects & Elevation

- Shadow scale (sm, md, lg, xl) for depth hierarchy
- Border radius scale (none, sm, md, lg, full)
- Hover and focus states
- Transition timing and easing
- Glassmorphism or other modern effects (if appropriate)

### 8. Accessibility Standards

- WCAG 2.1 AA compliance requirements
- Color contrast ratios (minimum 4.5:1 for text)
- Focus indicator styling
- Touch target sizes (minimum 44x44px)
- Reduced motion considerations

## Home Page Design Requirements

### Page Structure

Design a modern landing page with the following sections:

#### 1. Navigation Header
- Logo placement (left)
- Main navigation links: Projects, Developers, Investors, How It Works, Community
- Right side: Search icon, Sign In, "Get Started" CTA button
- Sticky on scroll behavior
- Mobile responsive hamburger menu

#### 2. Hero Section
**Primary Message:** Communicate the value proposition clearly
- Headline: Powerful, benefit-focused (not just feature-focused)
- Subheadline: Explain the three-way marketplace (creators ↔ developers ↔ investors)
- Primary CTA: "Post Your Idea" or "Get Started"
- Secondary CTA: "Browse Projects" or "Find Developers"
- Visual element: Consider abstract illustration, isometric graphics, or subtle animation showing connection/collaboration
- Trust indicators: "X projects funded" / "Y developers" / "Z investments made"

#### 3. User Type Selection / Value Props
Three equal columns or cards showing:

**For Idea Creators**
- Icon/illustration
- Headline: "Have an idea?"
- Brief value proposition (2-3 lines)
- CTA: "Share Your Vision"

**For Developers**
- Icon/illustration
- Headline: "Ready to build?"
- Brief value proposition (2-3 lines)
- CTA: "Find Projects"

**For Investors**
- Icon/illustration
- Headline: "Looking to invest?"
- Brief value proposition (2-3 lines)
- CTA: "Discover Opportunities"

#### 4. How It Works Section
Step-by-step visual flow showing the platform process:
1. Create your profile
2. Post or discover projects
3. Connect and collaborate
4. Build and fund together
5. Launch and succeed

Use numbered steps, connecting lines, or timeline visualization.

#### 5. Featured Projects Section
- Section headline: "Trending Projects" or "Featured Opportunities"
- Grid of 3-4 project cards showing:
  - Project image/thumbnail
  - Project title
  - Brief description (truncated)
  - Skills needed (as tags)
  - Funding progress bar
  - Team spots available
  - Creator avatar
- "View All Projects" link

#### 6. Platform Statistics / Social Proof
Impressive numbers displayed prominently:
- Total projects created
- Developers in network
- Funding facilitated
- Successful launches
- Countries represented

Use large typography with subtle animation on scroll.

#### 7. Testimonials / Success Stories
- 2-3 testimonial cards with:
  - Quote text
  - Person photo
  - Name and role
  - Company/project name
  - User type badge (creator/developer/investor)
- Consider carousel for mobile

#### 8. Featured Developers Section (Optional)
- "Top Developers" or "Available Now"
- Horizontal scroll or grid of developer cards
- Skills, ratings, availability status

#### 9. CTA Section
Strong call-to-action banner:
- Compelling headline about joining the community
- Email signup or "Get Started" button
- Background: gradient, pattern, or subtle illustration

#### 10. Footer
- Logo and tagline
- Navigation columns:
  - Platform: Projects, Developers, Investors, How It Works
  - Company: About, Blog, Careers, Press
  - Resources: Help Center, Documentation, API, Status
  - Legal: Terms, Privacy, Cookie Policy
- Social media links
- Newsletter signup
- Copyright and language selector

## Visual Style Direction

### Mood & Aesthetic
- **Modern SaaS**: Clean, spacious, professional
- **Startup Energy**: Dynamic, optimistic, growth-oriented
- **Tech-Forward**: Subtle technical elements without being cold
- **Human-Centered**: Show real people, collaboration, success

### Imagery Guidelines
- Use illustrations that show connection and collaboration
- Diverse representation in any photos or avatars
- Abstract tech patterns can be used as backgrounds
- Avoid overused stock imagery clichés

### Animation & Interaction
- Subtle micro-interactions on hover
- Smooth scroll animations (fade in on scroll)
- Progress indicator animations
- Loading state animations
- Avoid excessive or distracting motion

### Illustration Style
Recommend one of:
- Flat/geometric illustrations with brand colors
- Isometric 3D-style illustrations
- Line art with gradient accents
- Abstract shapes and patterns

## Technical Constraints

- Design must be implementable with **Tailwind CSS**
- Components will be built in **Vue 3** (Nuxt 4)
- Consider CSS custom properties for theming
- Design tokens should be exportable to CSS/JS
- Keep animation performant (prefer CSS transforms/opacity)
- Images should have defined aspect ratios for responsive handling

## Questions to Consider

1. Should the design lean more corporate/professional or startup/energetic?
2. What is the primary emotion users should feel? (Trust? Excitement? Possibility?)
3. Should there be a dark mode consideration from the start?
4. What illustration style best represents the brand?
5. How prominently should the three user types be differentiated visually?

## Success Criteria

The design should:
- Clearly communicate the platform's value in under 5 seconds
- Guide users to the appropriate path (creator/developer/investor)
- Build trust and credibility
- Feel modern and innovative without being trendy
- Be fully accessible and inclusive
- Scale well as features are added
- Maintain consistency across all components
